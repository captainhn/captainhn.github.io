Veröffentlicht am 2018-03-17 | | 浏览 次

在人工智能界有一种说法，认为机器学习是人工智能领域最能体现智能的一个分支。从历史来看，机器学习似乎也是人工智能中发展最快的分支之一。在二十世纪八十年代的时候，符号学习可能还是机器学习的主流，而自从二十世纪九十年代以来，就一直是统计机器学习的天下。不知道是否可以这样认为：从主流为符号机器学习发展到主流为统计机器学习，反映了机器学习从纯粹的理论研究和模型研究发展到以解决现实生活中的实际问题为目的的应用研究。 ——陆汝钤

机器学习概述

机器学习是什么

  • 人工智能的一个重要学科分支多领域交叉学科
  • 数据驱动，在数据上通过算法总结规律模式，应用在新数据上
    ——Image from http://branchandbound.net/
  • 研究计算机是怎样模拟人类的学习行为，以获取新的只是和技能，并重新组织已有的知识结构使之不断改善自身。
  • 计算机从数据中学习出规律和模式，以应用在新数据上做预测的任务
    • 互联网时代：巨大的数据量、超高的数据维度

      Since an early flush of optimism in the 1950’s. smaller subsets of artificial intelligence - first machine learning, then deep learning, a subset of machine learning - have created ever larger disruptions.

机器学习算法

针对不同问题：

聚类（无监督学习）：

  • 根据数据样本上抽取的特征，挖掘数据的关联模式
  • 相似用户挖掘/社区发现
  • 新闻聚类

分类问题（监督学习）：

  • 根据数据样本上抽取的特征，判定其属于有限个类别的哪一个
  • 文本情感的褒贬分析（类别：1、褒 2、贬）
  • 垃圾邮件的识别（类别：1、垃圾 2、正常）
  • 图像内容识别（类别：1、人类 2、汪星人 3、喵星人 4、其他）

回归问题（监督学习）：

  • 根据数据样本上抽取的特征，预测连续值的结果
  • 房价的具体值
  • 某一物体的得分

强化问题：

  • 研究如何基于环境而行动，以取得最大化的预期利益
  • 游戏的最高得分
  • 机器人最大化的完成任务
    ——From 周志华 《机器学习》

  • 回归：线性回归、多项式回归、岭回归、树模型回归…

  • 聚类：K-means 、层次聚类、密度聚类、GMM…

机器学习基本流程 与工作环节

机器学习的应用是围绕着数据与算法展开的。
数据 + 机器学习算法 = 预测模型

机器学习不同阶段与作用

  1. 数据预处理：
    • 数据采样、数据切分、特征抽取、特征选择、降维。
  2. 模型学习：
    • 超参选择、交叉验证、结果评估、模型选择、模型训练。
  3. 模型评估：
  • 分类、回归、排序评估标准。

    机器学习的评估方法

    关键：获得可靠的测试集数据（test set）

  • 留出法（hold-out）
  • 交叉验证法（cross validation）
  • 自助法（bootstrap）

留出法：直接将数据集D划分为两个互斥的集合，其中一个集合作为训练集S，其中一个作为测试集T，在S上训练出模型后，用T来评估其测试误差，作为其泛化误差的估计。

  • 一般用于工业级别
  • 保持数据的分布一致性（例如：分层采样）
  • 测试集不能太大，不能太小（例如：1/5~1/3）
  • 多次重复划分（例如：100次随机划分）

k折交叉验证：先将数据集D划分为k个大小相同的互斥子集。每个子集都尽可能保持数据的分布一致性，即从数据集D中通过分层采样取得，然后每次利用k-1个子集进行训练，剩下的子集进行测试。这样就可以获得K组训练/测试集，从而进行K组训练和测试，最终返回的是这K组的均值。

  • 多用于中小数据集。

——From 周志华《机器学习》

自助法：以自主采样法为基础，给定一个包含m个数据的数据集D，我们对他进行采样产生数据集D’：每次随机从数据集D中挑选一个样本，将其拷贝到数据集D’，然后将此数据放回到数据集D，为有放回采样。这个过程执行m次，我们就得到了包含m个样本的数据集D’。显然，D中的一部分数据将会在D‘中重复出现，而另一部分样本则不会出现。通过运算我们可以得到有36.8%的样本没有出现在数据集D’，于是我们可以将数据集D‘作为训练集，而D\D’（\：集合的减法）作为测试集。

  • 自助法在数据集较小、难以有效划分训练/测试集时很有用。

——From 周志华《机器学习》

机器学习的评估度量标准

关于模型的 “好坏” 的判断，不仅取决于算法和数据，还取决于当前的任务需求。

  • 性能度量（performance measure）是衡量模型泛化能力的数值评价标准，反应了当前问题（任务需求）
  • 使用不同的性能度量可能会导致不同的评判结果
